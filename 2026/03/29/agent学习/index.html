初识智能体

智能体是通过sensor(传感器)感知环境(Environment),并自主的通过Actuators（执行器）采取Action（行为），模仿人的感受与行为，连接到语言上就是属性与行为

环境是外部世界，通过摄像头、麦克风与雷达或者应用编程接口(Application Programming Interface，也就是API)返回的数据流

接收数据后要通过行为对环境施加影响，执行器既可以是物理设备（如机械臂与方向盘(感觉与智能驾驶有关)）或者虚拟工具(执行代码或者调用服务)

赋予其智能的是其自主性

一些远古(传统)智能体(从简单到复杂、从被动反应到主动学习)

反射智能体：由”条件-动作”规则构成，完全依赖感知输入，不具备记忆或预测能力

如果环境的当前状态不足以作为决策的全部依据，智能体该怎么办？

引出了

基于模型的反射智能体（Model-Based Reflex Agent）：引入了”状态”概念，其有一个内部的世界模型(World Model)用于追踪和理解环境中那些无法被直接感知的方面

仅仅理解世界还不够，智能体需要有明确的目标

基于目标的反射智能体(Goal-Basd Agent):其行为不再是被动的对环境作出反应，而是主动的、有预见性地导向某个特定未来状态的行动

经典的例子是 GPS 导航系统：你的目标是到达公司，智能体会基于地图数据（世界模型），通过搜索算法（如 A*算法）来规划（Planning）出一条最优路径。这类智能体的核心能力体现在了对未来的考量与规划上。

现实世界的目标往往不是单一的。我们不仅希望到达公司，还希望时间最短、路程最省油并且避开拥堵。

基于效用的智能体(Ulility-Based Agent):其目的变为最大化期望效率

以上均局限于设计师的经验。经验毕竟是局限的，如果让智能体不依赖预设而是要通过与环境的互动主动学习(一下对话并不贴合这个，只是闲的无聊问了一下)

大语言模型驱动的新范式

以GPT(Generative Pre-trained Transformer，原来GPT的全名是这个)

由大语言模型驱动的 LLM 智能体，其核心决策机制与传统智能体存在本质区别，从而赋予了其一系列全新的特性

这种转变来自于核心引擎、知识来源、交互方式等多个维度可见

大模型的工作方式

1.规划与推理:将高层任务划分成子任务

2.工具使用

3.动态修正:将用户反馈视为新的约束

分类的维度

1.决策模型分类(上文提到的模型)

2.基于时间与反应性(追求速度的反应性，追求最优解的规划性)

3.基于知识表示的分类：

(1)符号AI(操作根据逻辑规则，符号是人可读的实体(词语、概念)，有条理的、基于思考的)

(2)亚符号AI(从数据中学习到，但是有黑箱，逻辑上报答不加，快速的凭直觉的)

(3)神经符号AI(二者结合)

任务环境

用PAES模型来描述一个任务环境，分析其性能度量(Peformance)、环境(Environment)、执行器(Actuator)和传感器(Sensor)

环境是部分可观察的(无法一次获取全部信息)，这就要求智能体必须具备记忆和探索能力（尝试不同的查询日期)

行为结果并非总是确定的，根据结果的可预测性，环境分为确定性和随机性

环境中还可能存在其他行动者，从而形成多智能体(Multi-agent) 环境

运行机制

智能体完成任务是通过一个持续的循环与环境进行交互，这机制称为智能体循环

perception(感知) ———————–>Thought（思考）进行planning(规划)和Tool Selection—–>Action ->observation(观察，其实就是perception)
